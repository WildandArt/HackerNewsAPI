\hypertarget{md_README_autotoc_md29}{}\doxysubsection{1. Download the Collection JSON File}\label{md_README_autotoc_md29}

\begin{DoxyItemize}
\item First, ensure you have the Postman collection JSON file saved to your local machine. You can access the Postman collection file \href{Postman/HackerNewsAPI.postman_collection.json}{\texttt{ here}}. This file contains all the endpoints, requests, and configurations defined in the collection.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md30}{}\doxysubsection{2. Importing the Collection into Postman}\label{md_README_autotoc_md30}

\begin{DoxyEnumerate}
\item {\bfseries{Open Postman.}}
\item On the main interface, look for the {\bfseries{“\+Import”}} button. This is usually located on the top left side of the app.
\item Click on {\bfseries{“\+Import.\+”}}
\item In the import dialog that appears, choose the {\bfseries{“\+Upload Files”}} tab.
\item Click {\bfseries{“\+Choose Files”}} or drag the JSON file directly into the import area.
\item Navigate to {\ttfamily Hacker\+News\+API/\+Postman/\+Hacker\+News\+API.\+postman\+\_\+collection.\+json} and select it.
\item Once the file is selected, click {\bfseries{“\+Import”}} to load the collection into Postman.
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md31}{}\doxysubsection{3. Using the Imported Collection}\label{md_README_autotoc_md31}

\begin{DoxyItemize}
\item After importing, the collection will appear in your $\ast$$\ast$\char`\"{}\+Collections\char`\"{}$\ast$$\ast$ tab on the left sidebar.
\item Expand the collection to view all the requests, folders, and other configurations within it.
\item Click on any request to view its details, including the method, URL, headers, and body content.
\item You can now execute any of the requests by selecting them and clicking the {\bfseries{“\+Send”}} button.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md32}{}\doxysubsection{4. Editing and Customizing the Collection}\label{md_README_autotoc_md32}

\begin{DoxyItemize}
\item You can modify any request within the collection to suit your needs. This includes changing endpoints, parameters, headers, and body content.
